# 3. Active Cycle

## 3.1 The Facility Floor

The play area is divided into three columns:

| Column | Contents | Notes |
|---|---|---|
| Centre | Storage ring + Prep ring + Beamline hutches | Main play area |
| Right (180px) | Active Jobs & Rapid Review panel — live status of all in-flight and pending proposals | Read-only HUD, not a physical room |

A wall with a single door gap separates the **Prep Room** from the **Hutch**. The player must pass through the gap to carry samples between rooms. The door gap position is fixed; the player cannot carry samples through the wall.

## 3.2 Sample Workflow — The Core Loop

Each sample in a proposal batch follows this mandatory sequence:

| Step         | Action                                                                           | Location       |
| ------------ | -------------------------------------------------------------------------------- | -------------- |
| 1. COLLECT   | Walk to User NPC → [Space] pick up raw sample                                    | Prep Room      |
| 2. DEPOSIT   | Walk to Prep Table → [Space] deposit raw sample (auto-processes 1.0–2.0s)          | Prep Room      |
| 3. COLLECT   | Walk back to Prep Table → [Space] pick up prepped sample                         | Prep Room      |
| 4. ROUTE     | Carry prepped sample to the hutch whose technique matches the proposal           | Corridor       |
| 5. INTERLOCK | Stand inside the matching hutch → hold position (1.8s) with prepped sample       | Hutch entrance |
| 6. MEASURE   | Walk to the beamline's Control Room → [Space] start measurement (auto-runs 4.5s) | Control Room   |
| 7. COLLECT   | Walk back to Control Room → [Space] pick up result → sample complete             | Control Room   |

**Steps cannot be skipped.** The player can carry up to 3 samples in different stages simultaneously.

Stations always check for ready **OUTPUT** before accepting new input (pickup priority over deposit).

> **Technique routing (Step 4):** Each beamline is permanently assigned one technique. A prepped sample can only be taken to the hutch whose technique matches its proposal. Approaching the wrong hutch triggers an error message. See [4. Proposal System](4.%20Proposal%20System.md) § 4.3 for beamline assignment details.

## 3.3 Player Inventory

- Player can hold up to **3 samples** simultaneously
- Each held sample shown as a coloured dot above the player, labelled **R** (raw), **P** (prepped), or a lock icon (interlocked)
- Sample colour matches the job slot colour (blue/orange/green/purple/brown)
- Holding 3 samples does **not** prevent depositing at a station — only prevents collecting new samples from User NPCs

## 3.4 Stations

| Station | Room | Interaction | Base Duration | Capacity |
|---|---|---|---|---|
| Prep Table | Sample Prep Room | Deposit (Space) → auto-processes | 1.0–2.0s (random; = 0.5–1 game day) | 1 (upgradeable to 2) |
| Experimental Hutch | Hutch (must match proposal technique) | Player-held action (stand still with prepped sample) | 1.8s | 1 per beamline |
| Measurement Console | Control Room | [Space] start → auto-runs | 4.5s | 1 (upgradeable to 2) |

- A station with capacity > 1 processes multiple samples **in parallel** (not sequentially)
- Shows a progress bar and percentage label while processing
- Green ✅ READY indicator appears when output is waiting
- Amber stripe appears when the station is full

## 3.5 User NPCs

- NPCs for **all committed proposals** (including those queued behind the active slot limit) spawn at game start
- Up to **5 NPCs** can be present simultaneously (one per job slot)
- Positions are randomised inside the prep ring, avoiding the beamline room corridors (±28° around each beamline axis) and maintaining minimum spacing between NPCs
- NPCs bob gently; interaction ring glows bright when player is within pickup range
- NPC disappears once **all samples in its batch are fully processed** (not just collected)
- NPCs for queued proposals are visible but non-interactable until their slot becomes active

## 3.6 Active Jobs Panel (Right HUD)

Shows all in-flight jobs (up to 5 slots). Each slot displays:
- Job name, technique
- Samples done / total
- Current status (→ Needs prep / → Carry to Interlock / → Run the scan!)
- Pip progress bar

A `+N queued` label shows how many committed proposals are waiting for a slot to open.

## 3.7 Rapid Review Proposals

- Bonus proposals that appear in the **Active Jobs panel** (right column) during the active cycle
- Displayed with a ⚡ icon and orange card border; no "RAPID:" text prefix in the card
- No commitment penalty regardless of completion
- Visible in **any empty panel slot**, including slots beyond the current active-job limit — player sees the proposal even when all job slots are full (clicking shows a "slots full" error until one opens)
- **Expire after 40 seconds** if not accepted
- First rapid proposal appears ~20s into the cycle; subsequent ones appear at random intervals (10–20s)
