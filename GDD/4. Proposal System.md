# 4. Proposal System

## 4.1 Proposal Generation

Proposals are generated by combining an **Application Domain** with a **Technique** and a sample batch size. Each proposal has a reputation reward and an unfulfilled-commitment penalty (≈35% of rep value).

The penalty applies at cycle end if the player committed to the proposal but completed **0 samples**.

## 4.2 Application Domains (by Generation)

Application domains unlock progressively as the facility advances through eras. Later generations include more complex science with higher reputation rewards and larger sample batches.

Rep scales with batch size: `rep = base_rep + (samples − 1) × 8 ± small random`
Penalty = `round(rep × 0.35)`

> **Design note:** Base rep is intentionally flat across all generations (10–20 range). In later years, players earn more reputation through *volume* — running more proposals faster — rather than higher per-proposal rewards. This keeps the rep-per-cycle ceiling consistent while rewarding facility investment.

## 4.3 Techniques & Beamline Assignment

Each beamline (**BL-1 through BL-4**) is assigned one technique at the start of each year. The 4 techniques are **randomly drawn** from all techniques unlocked so far (see §7 — Facility Eras). The player will never receive a proposal for a technique that has no matching beamline.

### Technique Pool by Generation

| Gen | Techniques Added |
|---|---|
| 1st | Powder Diffraction, X-ray Fluorescence, Laue Diffraction, X-ray Topography |
| 2nd | EXAFS / XANES, SAXS, Protein Crystallography |
| 3rd | Micro-CT, Micro-XRF, XRD Analysis |
| 4th | Ptychography, Serial Crystallography, Phase Contrast, Time-Resolved XAS, Coherent Imaging |

From Gen 2 onward, the pool is larger than 4, so the yearly draw creates strategic variation — some techniques will not appear in a given year.

### Routing rule

A prepped sample can only enter the hutch of the beamline whose technique matches the proposal. Approaching the wrong hutch while holding a prepped sample shows an error: *"Wrong beamline! BL-X only runs [technique]"*. Once experiment setup is complete, the sample is locked to that beamline's measurement console.

## 4.4 Commitment & Penalty Rules

- The **Proposal Review screen** shows **8 cards** (4-column × 2-row grid) — the player may commit to any number of them
- At cycle start, the first committed proposal auto-activates (1 NPC enters immediately); remaining committed proposals appear as **click-to-start cards** in the Active Jobs panel
- The player can run **up to 5 concurrent active users** at once — additional committed proposals wait in queue
- Committed proposals incur a **−8 rep penalty** if 0 samples are completed by cycle end
- Proposals still in the commit queue at cycle end (never activated by the player) also incur the penalty
- **Rapid Review** proposals have **NO penalty** regardless of completion
- Partial completion (1+ samples but less than batch) earns proportional rep with **no penalty**

## 4.5 Publication Delay *(Planned — Not Yet Implemented)*

Completing a proposal triggers a **delayed reputation bonus** 2–5 cycles later, representing the time for the user to analyse data and publish. This range reflects real-world post-beamtime publication timelines of 6–18 months (each cycle ≈ 90 days). Publication probability and journal tier will depend on batch completion percentage.
