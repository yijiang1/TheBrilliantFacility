# 2. Scene Flow & Structure

## Scene Sequence

```
Boot → Proposal Review → Game (Active Cycle) → Cycle End / Upgrade Shop → Proposal Review → ...
```

After every 3rd cycle (year end), the Cycle End screen also awards new yearly funding before returning to Proposal Review.

## 2.1 Proposal Review Screen

Before each cycle, the player sees **6 randomly generated proposals** in a 2×3 card grid. Each card shows:
- Experiment name
- Technique
- Sample batch size
- Reputation reward
- Unfulfilled-commitment penalty (≈35% of rep value)

There is **no selection limit** — the player can commit to all 6. Only 3 run simultaneously; extras queue automatically and activate when a slot frees up.

The player can see their current funding balance on this screen. Clicking a card toggles its selection. Clicking **Start Cycle** begins the active game with all selected proposals pre-loaded as committed jobs.

## 2.2 Active Cycle (Game Scene)

The core 3-minute action phase. The cycle timer displays as game-days (90 → 0). The player moves their character around the facility floor, processes samples through the four-step workflow, manages station queues, accepts Rapid Review proposals, and responds to hazards.

See [[3. Active Cycle]] for full detail.

## 2.3 Cycle End / Upgrade Shop

After the timer expires, the Cycle End screen shows:
- Brief results summary (samples done, rep, ring stability, paper outcome, penalties)
- **Cycle Analytics panel** — proposals done / partial / failed; throughput in s/sample and rep/sample (auto-calculated from actual play data)
- **Between-cycle action choice** — player must choose one free action: 📄 Write Paper (+rep) or 🎓 Train Postdoc (level up AI). Continue button is locked until an action is chosen. See [§6.5](6. Progression.md).
- Upgrade shop — player spends accumulated funding on persistent upgrades

At the end of Cycle 3 (year end), new funding is automatically awarded.

Each Cycle End screen automatically appends a stat record to `localStorage` (see §9.4 Dev Tooling). Two dev buttons appear at the bottom-left: **Export Stats JSON** and **Clear Stats**.

See [[6. Progression]] for upgrade details.
